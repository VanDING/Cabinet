### B.1 Blackboard Mid-Session 推送 (原则 4, P0)

**问题**: 当前 Blackboard snapshot 仅在 AgentLoop 构建 system prompt 时注入一次（`agent-loop.ts:942-948`）。Agent A 写入 discovery 后，已在运行的 Agent B 感知不到。

**设计**:

```
Agent A write → BlackboardTopicRouter → EventBus SystemNotification
                                              │
Agent B 的 BlackboardObserver (新增) ← 订阅 ← ┘
  → ctx.pendingUpdates.push(entry)
  → 下一个 step 开始时, ContextBuilder 追加到 messages
```

**实施步骤**:

**B.1.1** 新建 `packages/agent/src/observers/blackboard-observer.ts`:

```typescript
import type { AgentObserver, AgentExecutionContext } from '../observer-pipeline.js';
import type { EventBus } from '@cabinet/events';
import { MessageType } from '@cabinet/types';

export class BlackboardObserver implements AgentObserver {
  name = 'BlackboardSync';
  private pendingUpdates: Array<{ topic: string; payload: unknown; agentId: string }> = [];

  constructor(
    private eventBus: EventBus,
    private watchedTopics: string[] = ['discoveries'],
  ) {
    this.eventBus.subscribe(MessageType.SystemNotification, (envelope) => {
      const data = (envelope.payload as any)?.data;
      if (data?.topic && this.watchedTopics.includes(data.topic)) {
        this.pendingUpdates.push({ topic: data.topic, payload: data.entry, agentId: data.agentId });
      }
    });
  }

  async onStepEnd(ctx: AgentExecutionContext): Promise<void> {
    if (this.pendingUpdates.length === 0) return;
    (ctx as any).pendingBlackboardUpdates = [...this.pendingUpdates];
    this.pendingUpdates = [];
  }
}
```

### B.2 PIS Intent Alignment Embedding 升级 (原则 2, P1)

**问题**: `process-identity-score.ts:50-68` 使用 Jaccard keyword overlap。语义相似但用词不同 → 误判 drift。

**前置条件检查**: `@cabinet/gateway` 的 `LLMGateway` 有 `generateEmbeddings` 方法。不存在独立的 `EmbeddingService` 类。需要创建一个薄封装或直接注入 LLMGateway。

**实施步骤**:

**B.2.1** 新建 `packages/agent/src/embedding-service.ts`:

```typescript
import type { LLMGateway } from '@cabinet/gateway';

export class EmbeddingService {
  constructor(private gateway: LLMGateway) {}

  async embed(text: string): Promise<number[]> {
    const result = await (this.gateway as any).generateEmbeddings({ input: [text] });
    return result.embeddings?.[0] ?? [];
  }

  async cosineSimilarity(a: string, b: string): Promise<number> {
    const [embedA, embedB] = await Promise.all([this.embed(a), this.embed(b)]);
    if (embedA.length === 0 || embedB.length === 0) return 0;
    const dot = embedA.reduce((sum, v, i) => sum + v * (embedB[i] ?? 0), 0);
    const magA = Math.sqrt(embedA.reduce((sum, v) => sum + v * v, 0));
    const magB = Math.sqrt(embedB.reduce((sum, v) => sum + v * v, 0));
    return dot / (magA * magB || 1);
  }
}
```

### B.4 Tool Variety 定量度量 (原则 7, P1)

**问题**: 系统不知暴露多少 tool、实际使用多少。原则 7 要求定量追踪 variety gap。

**B.4.1** 新建 `packages/agent/src/tool-variety-collector.ts`:

```typescript
export interface ToolVarietySnapshot {
  sessionId: string;
  exposedTools: number;
  usedTools: number;
  uniqueToolsPerStepAvg: number;
  gapRatio: number; // exposed / used
  topTools: [string, number][]; // top 5 by usage count
}

export function collectToolVariety(
  sessionId: string,
  toolCallHistory: { name: string }[],
  totalExposedTools: number,
): ToolVarietySnapshot {
  const usedNames = new Set(toolCallHistory.map((tc) => tc.name));
  const freq = new Map<string, number>();
  for (const tc of toolCallHistory) {
    freq.set(tc.name, (freq.get(tc.name) ?? 0) + 1);
  }
  return {
    sessionId,
    exposedTools: totalExposedTools,
    usedTools: usedNames.size,
    uniqueToolsPerStepAvg: toolCallHistory.length > 0 ? usedNames.size / toolCallHistory.length : 0,
    gapRatio: usedNames.size > 0 ? totalExposedTools / usedNames.size : Infinity,
    topTools: [...freq.entries()].sort((a, b) => b[1] - a[1]).slice(0, 5),
  };
}
```
