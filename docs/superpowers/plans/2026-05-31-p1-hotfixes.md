## Task 1: 给所有空 catch / `.catch(() => {})` 添加日志

**背景:** 全库 120+ 处静默吞错，导致本地调试无法定位问题层级。

**策略:** 不抛错、不改控制流，仅在 catch 块内加一行 `logger.warn('xxx failed', error)`。如果 catch 块已有日志或处理逻辑，跳过。

**Files:**
- Scan: 全库 `.ts`, `.tsx` 文件
- Modify: 所有包含空 catch 或 `.catch(() => {})` 的文件
- Test: `apps/server/src/__tests__/security-audit.test.ts`（如存在可扩展）或新增 `apps/server/src/__tests__/error-logging.test.ts`

---

- [ ] **Step 1: 扫描空 catch 位置**

Run:
```bash
rg "catch\s*\(\s*\)\s*\{\s*\}" --type ts -n
rg "\.catch\s*\(\s*\(\s*\)\s*=>\s*\{\s*\}\s*\)" --type ts -n
rg "catch\s*\([^)]*\)\s*\{\s*//\s*(TODO|FIXME|ignore|no-op)?\s*\}" --type ts -n
```
Expected: 输出所有匹配的文件和行号，保存到 `/tmp/empty-catches.txt`

---

- [ ] **Step 2: 确认日志工具可用**

Read: `packages/storage/src/logger.ts`
Verify: 存在 `logger.warn(message: string, ...meta: unknown[])` 或类似方法
If not found, use `console.warn` as fallback and note in commit message.

---

- [ ] **Step 3: 批量替换空 catch 为带日志版本**

For each file from Step 1, apply this exact transformation:

```typescript
// Before:
} catch {
}

// After:
} catch (error) {
  logger.warn('Operation failed', error);
}
```

And for promise catch:
```typescript
// Before:
.catch(() => {})

// After:
.catch((error) => { logger.warn('Async operation failed', error); })
```

**约束:**
- 如果 catch 块已有任何语句（即使是注释），不修改
- 如果文件没有导入 `logger`，添加：`import { logger } from '@cabinet/storage/logger';`（或相对路径，按该文件现有导入风格）
- 如果 logger 导入路径不确定，先用 `console.warn`

---

- [ ] **Step 4: 运行 TypeScript 编译检查**

Run: `pnpm run typecheck`（或 `pnpm tsc --noEmit`，按项目 package.json 中定义的命令）
Expected: 无新增编译错误

---

- [ ] **Step 5: 运行现有测试**

Run: `pnpm test`
Expected: 全部通过（不应因加日志而失败）

---

- [ ] **Step 6: Commit**

```bash
git add -A
git commit -m "chore: add logging to all empty catch blocks

Prevents silent failures during local development.
Each catch now logs via logger.warn with the caught error."
```
